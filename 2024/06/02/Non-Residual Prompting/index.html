Fine-Grained Controllable Text Generation Using Non-Residual Prompting

发表于 2024-06-02 分类于 深度学习 阅读次数： 本文字数： 1.6k 阅读时长 ≈ 6 分钟
现有的可控文本生成方法分为两种，一种是在每个解码步骤中通过修改token的概率分布来实现可控，限制了语言模型的规划能力。另一种是将控制指令融合到语言模型中，这使得语言模型在细粒度的控制方面表现的较差。为此作者提出了一种基于非残差注意力机制的语言模型NRP，NRP通过非残差注意力机制来处理上下文信息和经过编码器的提示信息来实现细粒度的控制。

问题

由于这种对可控文本生成的明显需求，最近的工作(见2.1节)探索了不同的方法来控制和约束CLM的生成过程。这方面的研究主要有两条线。较为传统的方法侧重于细粒度控制以及如何将生成过程控制在任意点，同时仍然坚持当前上下文。这往往是通过在每个解码步骤独立地修改预测的词汇分布来实现的。然而，这将CLM与控制方法解耦，禁止CLM进行相应规划的能力，从而严重限制了可制定的控制类型。

第二种方法转而选择更具表现力和更高层次的控制，让CLM本身解释并将指令纳入文本生成。这通常是通过一个微调的目标，或者像目前常见的那样，通过将指令制定为文本上下文(简称提示语)来完成的。虽然这些方法具有表现力，但在控制特定点的生成方面比以前的方法效果差。这是由于提示符的影响力与提示符到下一个预测令牌(邹薇等, 2021)的距离负相关，使得对不相邻文本的提示变得困难。

解决方法

为了弥合细粒度控制和提示表达能力之间的差距，我们提出了一种允许在整个生成过程中进行远程和独立提示的体系结构。该架构具有编码器-解码器设置，其中编码器通过一种新的非残差注意力模式影响解码器。除了对这种架构的好处进行理论论证之外，我们还提供了一种资源高效的自监督方法，用于将预训练的CLM转换为此设置。

模型

NRP模型引入了一个提示模型，即一个单独的编码器来对提示信息进行解释编码，并生成位置不变的键值对。然后CLM通过非残差注意力机制来处理提示信息。位置不变性保证了指令在任意时间步都是同等适用的，并通过其键值对的额外移位实现。

非残差注意力机制

为了在不同的时间步允许独立的提示，CLM计算了两个不同的信息流，分别为文本流和非残差流。文本流忽略提示模型，与CLM的正常自注意力相同，参与当前时间步和关注之前时间步的键值对。非残差流在文本流的基础上还关注提示模型的键值对并负责每个时间步的预测。如图所示：

结论

优点

本文引入了非残差注意力的概念，并论证了如何利用非残差注意力来控制生成文本模型。此外，该工作还指出了缺乏开放可控的文本生成任务，需要模型也考虑给定的上下文。作者试图通过引入人工创建的$C^2GEN$数据集，引入语境化的词语包含任务来弥补这一缺陷。

在$C^2GEN$和Common Gen上的实验结果清楚地表明，使用非残差提示模型增加了CLM的生成控制。与其他方法相比，该方法是最通用的，在所有测试的情况下都表现良好。
